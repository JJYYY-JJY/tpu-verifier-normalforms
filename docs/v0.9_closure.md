# v0.9 Closure: v6e Measured Baseline

This document freezes the current `main` baseline before the CertiNF-v6e
mainline refactor. The baseline tag is `v0.9-v6e-measured-baseline`.

## Scope

Implemented surfaces:

- Finite-field RREF over prime `F_p` with exact modular row operations,
  deterministic leftmost and min-fill teachers, replay, and RREF predicates.
- Fixed-shape RREF teacher trajectory NPZ shards for imitation learning.
- JAX/Flax/Optax `PivotMLP` training and Orbax checkpoints outside git.
- Verifier-guided RREF neural rollout with explicit failure accounting and no
  hidden teacher fallback.
- Compact RREF benchmark reporting for generated and shard-backed samples.
- Integer row-style HNF environment with exact unimodular row operations,
  exact coefficient-growth metrics, benchmark coverage, learned-policy v0.8
  training, DAgger, actor-critic, and verifier beam search.
- SNF certificate schema, exact Python replay/verifier, generated certificate
  benchmark coverage, and compact report integration.
- Lean checker workspace for small RREF JSON certificates over prime finite
  fields and SNF JSON certificates over exact integers. Lean remains
  checker-only.
- Paper-style benchmark report generation with Markdown, JSON metrics, and PNG
  plots from compact RREF/HNF/SNF summaries.
- Colab v6e measured RREF run notebook and compact measured artifacts under
  `results/measured/`.

## Exactness And Fallback Boundary

- Environments, certificate replay, algebraic predicates, and Lean checkers use
  exact integer or modular arithmetic.
- Floating-point tensors are confined to training and model scoring layers.
- Invalid matrices, non-prime finite-field moduli, malformed certificates, and
  illegal row/column operations must fail with explicit exceptions.
- Neural rollout failure is reported as neural failure. It must not be replaced
  by teacher output.
- Deterministic teachers are valid only as oracle, baseline, or dataset source.

large RREF rollout target.

## Known Bottlenecks

- Workload size is too small to saturate v6e.
- RREF rollout is sample-wise and host-heavy.
- `PivotMLP` predicts full traces rather than a stronger step policy with
  batched search.
- Benchmarking spends most wall time outside TPU tensor throughput.
- The next mainline should move toward backward-generated state shards,
  MatrixFormer-style single-step policies, device-batched beam/search, and CPU
  exact verification.
